# gnome-clipboard-history-native

Нативная история буфера обмена для GNOME на **X11** — как `Win+V` в Windows.
Резидентный демон на Go; по горячей клавише показывает у курсора попап со списком
записей (тема **Yaru**), стрелки двигают выбор, `Enter` вставляет выбранное в
активное окно, `Escape` закрывает.

> Статус: рабочий каркас на статичных данных (10+ демо-записей). Реальный захват
> истории буфера — в планах.

## Требования

- **X11**-сессия (не Wayland): нужны XTEST-инжект и override-redirect окна.
- GNOME (mutter) — горячая клавиша вешается через GNOME (mutter сам держит Super,
  приложение перехватить Super не может).
- Go 1.23+, `libgtk-3-dev`, cgo. Рантайм: GTK3, X-сервер с XTEST.

## Сборка

```sh
go build -o clipmgr .
```

Используется локальный форк `gotk3` в `third_party/gotk3` (через `replace` в
`go.mod`) — в апстримном v0.6.4 не хватает импорта, из-за чего он не собирается на
свежем Go; фикс — один добавленный импорт.

## Архитектура

- **Демон** (`clipmgr`): `gtk.Main()`, слушает unix-сокет `$XDG_RUNTIME_DIR/clipmgr.sock`.
- **Клиент** (`clipmgr --show`): «звонок» — его запускает GNOME-хоткей; будит демона.
- Гибрид **GTK + xgb**: окно и тема — GTK (`GTK_WINDOW_POPUP`); ввод — xgb
  (`GrabKeyboard` на root + поллинг), потому что у всплывшего GTK-окна GNOME
  отбирает фокус. Вставка — нативный XTEST; активное окно — EWMH; буфером владеет
  сам GTK-демон.

## Установка (вручную, пока без инсталлятора)

1. Собрать бинарник (см. выше).
2. Автозапуск демона: `~/.config/autostart/clipmgr.desktop` (Exec → путь к бинарнику).
3. Горячая клавиша: GNOME custom keybinding → `clipmgr --show`.
